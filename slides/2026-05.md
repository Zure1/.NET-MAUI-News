# Executive summary

May is mainly about **validation**.

- MAUI 10.0.70 is the stable update to test
- .NET 11 Preview 4 moves MAUI mobile to CoreCLR by default
- Preview workloads need newer platform tools
- Xcode 26.5 means Mac build hosts need version control

# Framework / runtime updates

- MAUI 10.0.70 shipped with many quality fixes
- Important areas:
  - CollectionView
  - CarouselView
  - Blazor Hybrid
  - Button / ImageButton
  - Windows WebView
- .NET 11 Preview 4 makes CoreCLR the default MAUI runtime on mobile

# CoreCLR: measure first

CoreCLR can help with:

- startup foundations
- diagnostics
- runtime alignment with the rest of .NET
- future NativeAOT work

But it can also change:

- startup time
- package size
- GC behavior
- reflection-heavy libraries
- Mono-specific assumptions

<!--
Do not assume faster. Measure Release builds on real devices.

# Tooling / platform updates

- MAUI 11 preview workload uses:
  - Xcode 26.4
  - JDK 21.0.8
  - Android API 36
- Apple released Xcode 26.5
- May .NET servicing shipped security and non-security fixes
- CommunityToolkit.Maui 14.1.1 fixes Popup, StatusBar, and iOS SpeechToText behavior
