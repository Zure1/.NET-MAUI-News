# Executive summary

June was a **servicing and regression-management month**.

- MAUI 10.0.71 is the key stable servicing update
- .NET 11 / MAUI 11 preview work should move into a separate validation lane
- Android, Apple, VS, and Rider tooling all need build-host attention
- Main production risks: Shell, HybridWebView/WebView, CollectionView performance, Hot Reload, visual regressions

<!--
Frame the month as operational hygiene: update deliberately, pin toolchains, and validate UI paths.
-->

---

# Top 3 must-discuss items

1. **MAUI 10.0.71 upgrade decision**  
   Patch value is high, but validate Shell, WebView, maps, keyboard, and tab/modal flows.

2. **Preview isolation**  
   .NET 11 / MAUI 11 preview lanes are useful now, but should not contaminate production CI.

3. **Build-host drift**  
   Android workload, Xcode, Java, SDK, and IDE versions need explicit pinning.

# Tooling / platform updates

- **Visual Studio 2022 17.14 servicing**: stability/security updates; reproduce MAUI problems with CLI to isolate IDE issues
- **Rider 2026.1.3**: bug-fix update; 2026.1 improves MAUI iOS remote Mac build-host workflow
- **.NET Android 36.1.69**: validate Android release builds, AAB, signing, and device deploy
- **Android Studio Quail 1**: Gradle parallel-sync behavior changed for large projects
- **Apple June releases**: Xcode 26.6 RC, iOS/macOS 26.6 betas, Xcode 27 beta

<!--
Focus discussion on reproducibility: everyone should know exactly which Xcode, SDK, Java, Android workload, and IDE version release builds use.

# Production risks and regressions

- **XAML Hot Reload broken reports**: developer-loop risk; verify with rebuild/redeploy
- **CollectionView performance regression**: GC churn and janky scrolling in complex cells
- **Android LinearGradientBrush regression**: visual-branding risk
- **Shell / modal / keyboard / tab navigation fixes**: retest custom navigation flows
- **HybridWebView/WebView Windows rendering churn**: retest hybrid screens before shipping

<!--
Turn these into release-checklist items, not passive watch items.

# Team action checklist

- Pin SDK via `global.json`
- Log `dotnet --info` and `dotnet workload list` in CI
- Pin Xcode / Java / Android SDK / workload versions
- Add non-blocking .NET 11 / MAUI 11 preview CI lane
- Smoke-test Shell, modal navigation, keyboard, HybridWebView, maps, CollectionView, CarouselView
- Profile CollectionView-heavy screens on physical devices
- Validate Android Release + AAB + signing + Play upload path
- Keep IDE/tooling issues reproducible through CLI where possible
