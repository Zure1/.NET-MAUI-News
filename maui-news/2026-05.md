## 1. .NET MAUI 10.0.70 stable servicing release

**What changed**

MAUI 10.0.70 was released on 21 May 2026. It includes many quality fixes. High-signal areas include:

- Blazor Hybrid APK asset handling
- Windows WebView close behavior
- Button and ImageButton visual fixes
- CollectionView fixes
- CarouselView fixes
- Android ItemsViewHandler cleanup

**Why it matters for MAUI developers**

These are common production controls and app paths. Apps with lists, carousels, Blazor Hybrid, image buttons, custom button states, or Windows targets should care.

**Recommended action**

Upgrade first in staging. Run focused UI checks around:

- CollectionView and CarouselView
- Blazor Hybrid
- Button and ImageButton states
- Windows WebView
- Android list screens

**Source link**

- https://github.com/dotnet/maui/releases/tag/10.0.70

## 2. .NET 11 Preview 4: CoreCLR becomes default for MAUI mobile

**What changed**

Starting with .NET 11 Preview 4, MAUI apps use **CoreCLR by default** on Android, iOS, Mac Catalyst, and tvOS. Mono can still be selected during the transition with:

```xml
<PropertyGroup>
  <UseMonoRuntime>true</UseMonoRuntime>
</PropertyGroup>
```

Microsoft also calls out that some larger Android apps have reported worse startup time or larger package size.

**Why it matters for MAUI developers**

This is a major runtime change. It can improve startup foundations and diagnostics, but it can also change app size, startup behavior, GC behavior, native packaging, and compatibility with libraries that use reflection or Mono-specific behavior.

**Recommended action**

Create a separate .NET 11 preview lane. Compare against your .NET 10 baseline:

- cold start and warm start
- APK/AAB/IPA size
- memory use
- navigation flows
- native integrations
- third-party libraries
- reflection and dynamic code paths
- diagnostics and profiling

Do not assume CoreCLR is faster for your app. Measure it.

**Source link**

- https://devblogs.microsoft.com/dotnet/dotnet-maui-moves-to-coreclr-in-dotnet-11/

## 3. .NET 11 Preview 4 and MAUI 11 preview workload

**What changed**

.NET 11 Preview 4 shipped in May 2026. The MAUI 11 preview workload was also published with workload set:

```bash
11.0.100-preview.4.26254.1
```

The MAUI preview release recommends:

- Xcode 26.4
- Java JDK 21.0.8
- Android API 36

**Why it matters for MAUI developers**

This affects CI images, Mac build hosts, Android SDK setup, workload install scripts, and local developer machines. Preview workloads can change build behavior quickly.

**Recommended action**

Do not mix preview workloads into production CI. Add a non-blocking preview job that installs the exact workload set and records:

- build failures
- publish failures
- trimming and AOT failures
- Android packaging issues
- iOS/Mac Catalyst signing issues
- device-test failures

**Source links**

- https://devblogs.microsoft.com/dotnet/dotnet-11-preview-4/
- https://github.com/dotnet/maui/releases/tag/11.0.0-preview.4.26230.3

## 4. May .NET servicing update

**What changed**

Microsoft released the May 2026 .NET servicing update on 12 May. It includes security and non-security fixes for .NET 10, .NET 9, and .NET 8.

**Why it matters for MAUI developers**

MAUI builds depend on SDK/runtime servicing and workload resolution. Hosted CI images can change even when app code does not.

**Recommended action**

Pin and log the toolchain:

- use `global.json`
- log `dotnet --info`
- log `dotnet workload list`
- update hosted CI images intentionally
- avoid unplanned SDK drift in release branches

**Source link**

- https://devblogs.microsoft.com/dotnet/dotnet-and-dotnet-framework-may-2026-servicing-updates/

## 5. Apple tooling and OS updates

**What changed**

Apple released Xcode 26.5 on 11 May 2026, alongside iPadOS, tvOS, visionOS, and watchOS 26.5 updates.

**Why it matters for MAUI developers**

MAUI iOS and Mac Catalyst builds are sensitive to Xcode selection. A Mac build-host update can change compile behavior, simulator behavior, signing, TestFlight upload, or App Store submission behavior.

**Recommended action**

Pin Xcode for release builds. Test Xcode updates in a staging lane before changing hosted runners or local build hosts. Add this to CI logs:

```bash
xcodebuild -version
```

**Source link**

- https://developer.apple.com/news/releases/

## 6. CommunityToolkit.Maui 14.1.1

**What changed**

CommunityToolkit.Maui 14.1.1 shipped in May. It includes fixes for Popup, StatusBar, and iOS SpeechToText behavior. It also raises the dependency to Microsoft.Maui.Controls 10.0.60.

**Why it matters for MAUI developers**

This is relevant if your app uses Toolkit Popup, StatusBar, or SpeechToText. The dependency bump also means Toolkit updates may move your app closer to MAUI 10.0.60+ behavior.

**Recommended action**

Upgrade in a branch first. Test:

- popup navigation lifecycle
- status bar color and orientation behavior
- iOS SpeechToText timeout behavior
- package dependency resolution

**Source link**

- https://github.com/CommunityToolkit/Maui/releases/tag/14.1.1
