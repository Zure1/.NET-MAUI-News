# .NET MAUI News — May 2026

**Date range covered:** 01. 05 -> 31. 05  
**Audience:** experienced .NET MAUI developers  
**Goal:** short briefing for a senior developer meeting.

## 30-second summary

May was mainly about **MAUI 10 servicing and regression cleanup**.

The biggest stable update was **.NET MAUI 10.0.70**. It fixed many UI and platform issues, but it also followed a risky 10.0.60 cycle where several regressions were reported. Teams should upgrade with focused regression tests, not as a blind patch.

## 1. .NET MAUI 10.0.70 shipped

**What changed**  
Microsoft released **.NET MAUI 10.0.70** on **21 May 2026**. The release notes say it includes **135 commits** focused on quality, performance, and developer experience.

Important areas:

- Blazor Hybrid APK asset handling
- Button and ImageButton fixes
- CollectionView and CarouselView fixes
- Android, iOS, Mac Catalyst, and Windows UI fixes
- Accessibility and visual-state fixes

**Why it matters**  
This is the main stable MAUI update for May. It touches controls that many production apps use every day.

**Recommended action**  
Upgrade first in a staging branch. Test:

- Shell navigation
- CollectionView and CarouselView screens
- Blazor Hybrid apps
- Android release builds
- Windows UI behavior
- custom visual states

**Source link**  
https://github.com/dotnet/maui/releases/tag/10.0.70

## 2. MAUI 11 Preview 4 arrived with new workload requirements

**What changed**  
Microsoft released **.NET MAUI 11.0.0-preview.4.26230.3** on **13 May 2026**.

The preview workload uses:

- Workload set: `11.0.100-preview.4.26254.1`
- Xcode: `26.4`
- Java JDK: `21.0.8`
- Android API: `36`
- Windows App SDK: `1.8.251106002`
- WebView2: `1.0.3179.45`

It also included early .NET 11 changes, including Blazor Hybrid template updates, Editor command work, and iOS/Mac map fixes.

**Why it matters**  
This preview is not for production, but it tells teams what build-host changes are coming. It is useful for finding migration problems early.

**Recommended action**  
Create a separate preview CI lane. Do not mix preview workloads into production pipelines.

Track failures by type:

- workload install
- Xcode or Android SDK mismatch
- linker/trimming/AOT
- third-party NuGet compatibility
- source or behavior changes

**Source link**  
https://github.com/dotnet/maui/releases/tag/11.0.0-preview.4.26230.3

## 3. Several 10.0.60 regressions were active in May

**What changed**  
After the 10.0.60 release at the end of April, several May issues were marked as regressions or high-priority problems.

Important examples:

| Area | Issue | Practical risk |
|---|---|---|
| Android drawing | `LinearGradientBrush` renders incorrectly | Branded UI can look broken |
| Shell | `Shell.NavBarIsVisible="False"` ignored | Navigation UI can appear when it should be hidden |
| Android WebView | WebView corrupts dark theme colors | Dialogs and app theme can look wrong |
| App icon build | Empty but valid SVG app icon can fail build | CI/release builds can break |
| Visual states | Custom controls can stay stuck in `Selected` state | Selection UI can become wrong across platforms |

**Why it matters**  
These are not edge-case problems for framework authors only. They affect app visuals, navigation, build output, and reusable custom controls.

**Recommended action**  
Before moving from 10.0.60 or 10.0.70, add targeted smoke tests for:

- gradients and branded UI
- Shell navigation bar visibility
- WebView in dark mode
- app icon generation in CI
- selected-state custom controls

**Source links**  
- https://github.com/dotnet/maui/issues/35280
- https://github.com/dotnet/maui/issues/35249
- https://github.com/dotnet/maui/issues/35270
- https://github.com/dotnet/maui/issues/35293
- https://github.com/dotnet/maui/issues/35399

## 4. CollectionView had important May issues and fixes

**What changed**  
A May issue reported that `CollectionView.ScrollTo(index)` does not work correctly when `IsGrouped="True"` on iOS, Mac Catalyst, and Windows. A related Android backport PR fixed a grouped `ScrollTo` regression for the servicing branch.

**Why it matters**  
CollectionView is one of the most important controls in production apps. Grouped lists are common in settings, inboxes, dashboards, and catalog screens. Scroll bugs are visible to users and easy to miss in unit tests.

**Recommended action**  
Add UI tests or manual checks for:

- grouped CollectionView scrolling
- `ScrollTo`
- status-bar tap-to-top behavior on iOS
- item selection after scrolling
- large lists on physical devices

**Source links**  
- https://github.com/dotnet/maui/issues/35326
- https://github.com/dotnet/maui/pull/35428
- https://github.com/dotnet/maui/issues/35420

## 5. Android WebView memory leak fix was merged

**What changed**  
A May issue reported that Android WebView file chooser callbacks could stay rooted through `ActivityResultCallbackRegistry`. A PR changed callback handling so callbacks are consumed once and removed before invocation.

**Why it matters**  
This matters for apps that use WebView file upload flows. Repeated file chooser use could keep objects alive longer than expected.

**Recommended action**  
If your app uses Android WebView file upload, test repeated open/cancel/upload flows after taking the servicing update that contains the fix. Watch memory with Android Studio Profiler or `dotnet-counters` where applicable.

**Source links**  
- https://github.com/dotnet/maui/issues/35405
- https://github.com/dotnet/maui/pull/35436

## 6. iOS and Mac navigation back-button fixes moved into servicing

**What changed**  
A May backport PR fixed `OnBackButtonPressed` not being invoked for `NavigationPage` and Shell on iOS and Mac.

**Why it matters**  
Apps that depend on back-button interception can break user flows, unsaved-change prompts, analytics, or custom navigation rules.

**Recommended action**  
Retest back navigation on iOS and Mac Catalyst, especially:

- Shell pages
- modal navigation
- unsaved-change dialogs
- custom back-button handling
- analytics or telemetry triggered during navigation

**Source link**  
https://github.com/dotnet/maui/pull/35434

## 7. CommunityToolkit.Maui 14.1.1 shipped

**What changed**  
CommunityToolkit.Maui **14.1.1** was released on **07 May 2026**.

Main changes:

- new Popup navigation helper for `NavigatingFromEventArgs`
- iOS SpeechToText auto-stop timeout fix
- StatusBar fixes
- `StatusBarBehavior` orientation handling
- dependency raised to `Microsoft.Maui.Controls` `10.0.60`
- source-generator improvement for BindableProperty default values

**Why it matters**  
This affects teams using Toolkit Popup, StatusBar, SpeechToText, or custom controls with generated bindable properties.

**Recommended action**  
Upgrade in a feature branch if you use these Toolkit features. Test popup navigation, status bar color/height, iOS speech timeout, and Android API 36 layout behavior.

**Source link**  
https://github.com/CommunityToolkit/Maui/releases/tag/14.1.1

## 8. Apple platform updates landed on 11 May

**What changed**  
Apple published May platform updates, including iOS/iPadOS/macOS/tvOS/watchOS/visionOS 26.5 releases on **11 May 2026**.

**Why it matters**  
MAUI iOS and Mac Catalyst builds depend heavily on Xcode and Apple SDK selection. Even small platform updates can affect simulators, signing, TestFlight, and SDK behavior.

**Recommended action**  
Pin Xcode on build agents. Keep one test lane for newer Apple SDKs, but keep production release builds on the Microsoft-supported Xcode/workload combination.

**Source link**  
https://developer.apple.com/news/releases/
