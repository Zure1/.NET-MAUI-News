# .NET MAUI News — May–June 2026

**Date range covered:** 01. 05 → 26. 06  
**Audience:** experienced .NET MAUI developers  
**Signal level:** only changes that affect building, testing, shipping, or maintaining MAUI apps.

## 30-second summary

The main theme is **safe upgrades and toolchain control**.

- **MAUI 10.0.70 and 10.0.71** are the stable updates to test.
- **.NET 11 Preview 4/5** matters because MAUI mobile moves toward **CoreCLR by default**.
- **Xcode, Android SDK, JDK, .NET SDK, and workloads** need stronger pinning in CI.
- Main watch areas: **CollectionView, Shell/navigation, WebView/HybridWebView, Hot Reload, gradients, and visual regressions**.
- **CommunityToolkit.Maui** shipped useful fixes for Popup, Camera, MediaElement, StatusBar, SpeechToText, DrawingView, and accessibility.

## 1. .NET MAUI 10.0.70 stable servicing release

**What changed**

MAUI 10.0.70 was released on 21 May 2026. Important fixes touched Blazor Hybrid APK assets, Windows WebView close behavior, Button/ImageButton visuals, CollectionView, CarouselView, Android ItemsViewHandler cleanup, accessibility, and drawing.

**Why it matters for MAUI developers**

These are common production areas. Apps with lists, carousels, Blazor Hybrid, custom buttons, drawing, or Windows targets should test this release carefully.

**Recommended action**

Stage the upgrade. Test CollectionView, CarouselView, Blazor Hybrid pages, Button/ImageButton states, Windows WebView, Android list screens, and custom drawing.

**Source link**

- https://github.com/dotnet/maui/releases/tag/10.0.70

## 2. .NET MAUI 10.0.71 stable servicing release

**What changed**

MAUI 10.0.71 landed in June as a targeted servicing update. Important fixes include Windows WebView/HybridWebView blank rendering, Windows FontImageSource centering, iOS/Mac Catalyst map retention, iOS Shell modal keyboard layout, Android BottomNavigationView visibility, SafeArea listener handling in recycler items, and Shell modal/tab navigation behavior.

**Why it matters for MAUI developers**

This touches navigation, hybrid screens, maps, keyboard behavior, tabs, and Windows rendering. These areas often break real production flows.

**Recommended action**

Validate before production: Shell navigation, modal navigation, Android bottom tabs, iOS keyboard layout, Windows WebView/HybridWebView, maps, and FontImageSource icons.

**Source link**

- https://github.com/dotnet/maui/releases/tag/10.0.71

## 3. .NET 11 Preview 4: CoreCLR becomes default for MAUI mobile

**What changed**

Starting with .NET 11 Preview 4, MAUI apps use CoreCLR by default on Android, iOS, Mac Catalyst, and tvOS. Mono can still be selected during the transition. Microsoft also notes reports where some larger Android apps saw worse startup time or larger package size.

**Why it matters for MAUI developers**

This is a major runtime change. It can affect startup, memory, package size, GC behavior, reflection, native integration, diagnostics, and third-party library compatibility.

**Recommended action**

Add a separate .NET 11 preview lane. Compare against your .NET 10 baseline: cold start, warm start, APK/AAB/IPA size, memory, navigation, native integrations, reflection-heavy code, crash reporting, and diagnostics.

**Source link**

- https://devblogs.microsoft.com/dotnet/dotnet-maui-moves-to-coreclr-in-dotnet-11/

## 4. .NET 11 Preview 4 and Preview 5

**What changed**

.NET 11 Preview 4 shipped in May. .NET 11 Preview 5 shipped on 9 June 2026 and includes runtime, SDK, library, C#, ASP.NET Core, EF Core, and MAUI updates. MAUI-related Preview 5 items include platform-fix work, cancellation-token-aware animation overloads, Windows Maps backed by Azure Maps, and .NET for Android API 37 stabilization work.

**Why it matters for MAUI developers**

This is migration preparation, not production adoption. It can expose future runtime, linker, trimming, AOT, Android, Apple, Windows, and NuGet compatibility issues.

**Recommended action**

Keep preview CI separate from production CI. Track compile breaks, workload install issues, packaging problems, signing issues, trimming/AOT failures, startup/app-size changes, and third-party NuGet problems.

**Source links**

- https://devblogs.microsoft.com/dotnet/dotnet-11-preview-4/
- https://devblogs.microsoft.com/dotnet/dotnet-11-preview-5/
- https://github.com/dotnet/maui/releases/tag/11.0.0-preview.4.26230.3

## 5. May and June .NET servicing updates

**What changed**

Microsoft released .NET servicing updates in May and June 2026. The June update includes security and non-security fixes for .NET 10, .NET 9, and .NET 8.

**Why it matters for MAUI developers**

MAUI builds depend on SDK/runtime servicing and workload resolution. Hosted CI images can change even when app code does not.

**Recommended action**

Use `global.json`, log `dotnet --info`, log `dotnet workload list`, pin workloads where needed, and update CI images intentionally.

**Source links**

- https://devblogs.microsoft.com/dotnet/dotnet-and-dotnet-framework-may-2026-servicing-updates/
- https://devblogs.microsoft.com/dotnet/dotnet-and-dotnet-framework-june-2026-servicing-updates/

## 6. Apple, Android, and platform tooling drift

**What changed**

Apple released Xcode 26.5 in May, iOS/macOS 26.5.1 in June, App Store Connect API 4.4, iOS/iPadOS/macOS 26.6 betas, Xcode 27 beta 2, Xcode 26.6, and TestFlight updates. Android tooling also moved, and MAUI 11 preview expects Xcode 26.4, JDK 21.0.8, and Android API 36.

**Why it matters for MAUI developers**

MAUI iOS and Mac Catalyst builds are sensitive to Xcode. Android release builds are sensitive to JDK, Android SDK/API level, packaging, signing, and hosted runner state.

**Recommended action**

Pin Xcode, JDK, Android SDK/API, .NET SDK, and workloads in CI. Log `xcodebuild -version` and Android SDK state. Keep beta Apple SDK and preview Android API testing separate from production release lanes.

**Source links**

- https://developer.apple.com/news/releases/
- https://developer.android.com/studio/releases

## 7. CommunityToolkit.Maui updates

**What changed**

CommunityToolkit.Maui shipped useful updates during this range:

- 14.1.1: Popup navigation helper, StatusBar fixes, iOS SpeechToText timeout fix.
- 14.2.0: Popup v2 Android screen-reader accessibility and Windows DrawingView rendered line-cap fix.
- MediaElement 10.0.0: stream media source support.
- Camera 6.1.0: `IsTorchOn` and Windows camera usage when microphone permission is denied.

**Why it matters for MAUI developers**

These updates matter for apps using popups, accessibility, media playback, camera, drawing, status bar control, or speech input.

**Recommended action**

Upgrade Toolkit packages in a branch. Test Popup lifecycle/accessibility, StatusBar color/orientation, iOS SpeechToText timeout, Windows camera permission-denied paths, torch behavior, MediaElement stream lifetime, and DrawingView output.

**Source link**

- https://github.com/CommunityToolkit/Maui/releases

## 8. Active regression watch

**What changed**

Worth tracking:

- XAML Hot Reload broken reports in some setups.
- CollectionView scrolling performance regression reports with more GC and jank.
- Android LinearGradientBrush regression reports.
- Shell/navigation and WebView/HybridWebView fixes in servicing releases.

**Why it matters for MAUI developers**

CollectionView and visual regressions can block releases. Hot Reload issues slow development and can hide whether a problem is tooling or runtime behavior.

**Recommended action**

Before release, run physical-device scroll tests, screenshot checks for gradients/icons/branded controls, Shell/modal/tab navigation smoke tests, and Windows WebView/HybridWebView smoke tests. Use full rebuild/redeploy before filing UI bugs.

**Source links**

- https://github.com/dotnet/maui/issues/35787
- https://github.com/dotnet/maui/issues/35344
- https://github.com/dotnet/maui/issues/35280
- https://github.com/dotnet/maui/releases/tag/10.0.71

## 9. Visual Studio and Rider tooling

**What changed**

Visual Studio 2022 17.14 servicing updates and Rider 2026.1.3 bug-fix updates landed during this range. I did not find a separate, high-signal MAUI-specific IDE feature that should be a main action item beyond normal servicing and build-host control.

**Why it matters for MAUI developers**

IDE updates can still affect XAML editing, Hot Reload, debugger behavior, device deployment, and workload installation.

**Recommended action**

When a release build fails, reproduce with CLI builds first. Record IDE version, SDK version, workload list, Xcode, JDK, and Android SDK state when filing bugs.
