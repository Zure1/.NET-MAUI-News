## 1. .NET MAUI 10.0.80 stable servicing release

**What changed**

.NET MAUI 10.0.80 was released on 24. 06. 2026. The release notes describe 112 commits focused on quality, performance, and developer experience.

Notable production areas touched:

- CollectionView and CarouselView spacing, scrolling, empty state, header, footer, and event behavior.
- Android, iOS, Mac Catalyst, and Windows control fixes for Button, Entry, Editor, DatePicker, TimePicker, Flyout, FlyoutPage, GraphicsView, and dialogs.
- Windows password Entry crash fix.
- Mac Catalyst DatePicker opened/closed event fix.
- Android Material 3 dialog styling fix.
- Essentials geolocation support for mean sea level altitude on Android API 34+.
- HybridWebView Android `message` event filtering by source.

**Recommended action**

- Move one staging branch to 10.0.80 and run UI regression tests for lists, carousels, Shell/navigation, Flyout, forms, dialogs, HybridWebView, and Windows packaging.
- If you are pinned to 10.0.70 or 10.0.71, test moving directly to 10.0.80.
- Pay special attention to any app code that depends on exact CollectionView scroll position, CarouselView events, Android dialogs, or Windows text-entry behavior.

**Source link**

- [dotnet/maui 10.0.80 release notes](https://github.com/dotnet/maui/releases/tag/10.0.80)

## 2. .NET MAUI 10.0.71 targeted June patch

**What changed**

.NET MAUI 10.0.71 was released on 10. 06. 2026 as a targeted patch after 10.0.70. The patch included fixes and reverts for:

- Windows WebView / HybridWebView blank rendering.
- Windows FontImageSource centering regression.
- iOS and Mac Catalyst MapPool retention with MapElements.
- iOS Shell modal keyboard layout after updating to 10.0.60.
- Android BottomNavigationView staying visible for TabbedPage inside modal NavigationPage.
- SafeArea listener handling in recycler items.
- Shell navigation lifecycle behavior after modal pop and tab changes.

**Recommended action**

- Treat 10.0.71 mainly as context for teams that adopted 10.0.70 early.
- Use 10.0.80 as the preferred June target unless you are bisecting a regression between 10.0.70, 10.0.71, and 10.0.80.
- Regression-test WebView/HybridWebView, Shell modal flows, Maps, FontImageSource, and Android tab navigation if you moved through 10.0.70 or 10.0.71.

**Source link**

- [dotnet/maui 10.0.71 release notes](https://github.com/dotnet/maui/releases/tag/10.0.71)

## 3. .NET 11 Preview 5 MAUI update

**What changed**

.NET 11 Preview 5 was announced on 09. 06. 2026. The MAUI release notes describe a large reliability wave plus platform and API work.

High-signal MAUI changes:

- Reliability fixes across CollectionView, CarouselView, Shell, gestures, layout, RTL / FlowDirection, SearchHandler, TabBar, Label, GraphicsView, SwipeView, HybridWebView, Entry, Editor, Picker, BoxView, ImageButton, RadioButton, navigation lifecycle, and ControlTemplate scenarios.
- `FadeToAsync`, `RotateToAsync`, `TranslateToAsync`, `ScaleToAsync`, `LayoutToAsync`, and related animation APIs now support `CancellationToken`.
- `BoxView.Fill` now supports `Brush`, including gradients and patterns.
- Windows `Map` now has a real WinUI MapControl implementation backed by Azure Maps instead of a throwing stub.
- Material 3 Android handlers and helpers are public.
- Toolbar back buttons now support screen-reader labels.
- .NET MAUI in .NET 11 raises Android minimum support to API 24.
- .NET for Android stabilizes API 37 / Android 17 and defaults .NET 11 projects to `net11.0-android37`.
- Android trimmable typemap work progressed for CoreCLR and NativeAOT.
- Apple workloads refreshed bindings for Xcode 26.5 and improved dev-loop builds.
- `Microsoft.Maui.Essentials.AI` Apple Intelligence APIs moved into the .NET 11 release branch, still behind the experimental warning.

**Recommended action**

- Keep .NET 11 work in a preview branch; do not move production apps until your SDK, IDE, CI, signing, store, and library stack support the target.
- For Android, update project assumptions around `SupportedOSPlatformVersion` to API 24 before upgrading to .NET 11.
- For Windows Maps, budget work for Azure Maps key configuration and test unimplemented cross-platform map features on Windows.
- Start replacing older animation calls with the `*Async` methods where cancellation is useful.
- Measure app size, startup, trimming, AOT, and crash reporting separately for Android and Apple targets.
- Use Visual Studio 2026 Insiders for .NET 11 validation on Windows, or use SDK-driven builds in isolated CI agents.

**Source links**

- [.NET 11 Preview 5 announcement](https://devblogs.microsoft.com/dotnet/dotnet-11-preview-5/)
- [.NET MAUI in .NET 11 Preview 5 release notes](https://github.com/dotnet/core/blob/main/release-notes/11.0/preview/preview5/dotnetmaui.md)
- [dotnet/maui 11.0.0-preview.5.26304.4 release](https://github.com/dotnet/maui/releases/tag/11.0.0-preview.5.26304.4)

## 5. .NET 8 and .NET 9 support deadline

**What changed**

Microsoft announced on 29. 06. 2026 that .NET 8 and .NET 9 both reach end of support on 10. 11. 2026. After that date, Microsoft will no longer provide servicing updates, security fixes, or technical support for those versions. The recommended supported target is .NET 10 LTS, which is supported through November 2028.

**Recommended action**

- Inventory MAUI apps still targeting `net8.0-*` or `net9.0-*`.
- Move active apps to .NET 10 and validate workload versions, NuGet dependencies, Android/iOS signing, MSIX packaging, and store submission pipelines.
- Avoid leaving production store apps on unsupported target frameworks after 10. 11. 2026.
- Remove out-of-support components from Visual Studio installations after your projects are migrated.

**Source link**

- [.NET 8 and .NET 9 will reach End of Support on November 10, 2026](https://devblogs.microsoft.com/dotnet/dotnet-8-9-end-of-support/)

## 6. CommunityToolkit.Maui June releases

**What changed**

CommunityToolkit shipped several MAUI updates on 02. 06. 2026:

- `CommunityToolkit.Maui` 14.2.0 fixed Popup v2 Android screen-reader behavior and Windows DrawingView exported/rendered line caps.
- `CommunityToolkit.Maui.MediaElement` 10.0.0 added `MediaSource.FromStream(Stream)` for stream-backed media playback.

- Regression-test Popup focus and screen-reader behavior on Android.

- Test CameraView permission combinations, especially Windows camera without microphone permission.

## 7. SkiaSharp 4 stable release

**What changed**

SkiaSharp 4.148.0 was announced on 29. 06. 2026 as the first stable SkiaSharp v4 release. The announcement says SkiaSharp sits under .NET MAUI and includes:

- Skia engine current through milestone m148.
- Rendering, codec, performance, and security work from upstream Skia.
- Variable font axis control and color font palettes.
- Animated WebP encoding.
- Retired legacy APIs and a cleaned-up API surface.
- Object lifecycle fixes for native reference-counted singletons.

**Recommended action**

- Evaluate SkiaSharp 4 in a branch if your MAUI app uses custom drawing, charts, image processing, maps, signatures, PDF/image export, or third-party controls that depend on SkiaSharp.
- Test graphics output across Android, iOS, Mac Catalyst, and Windows before upgrading production packages.
- Watch for removed legacy APIs and transitive dependency constraints from UI libraries.

**Source link**

- [SkiaSharp 4.0 is here: announcing the first stable release](https://devblogs.microsoft.com/dotnet/skiasharp-4-0-stable/)

## 8. Windows package identity and WinApp CLI

**What changed**

Microsoft published guidance on 29. 06. 2026 for using WinApp CLI with .NET desktop apps. The tool can initialize package identity, support `dotnet run` with identity, create MSIX packages with `winapp pack`, generate and install certificates, and expose Store-related tooling.

**Recommended action**

- Treat this as a Windows packaging tool to evaluate for special MAUI Windows scenarios, not as a replacement for a working MAUI packaging pipeline.
- Consider it if you need deeper local testing around package identity, Windows APIs, notifications, file handlers, background tasks, MSIX generation, or Store automation.
- Validate output packages against your existing signing, MSIX, Store, and enterprise deployment requirements before adopting it.

**Source link**

- [Packaging and Package Identity for .NET apps with WinApp CLI on Windows](https://devblogs.microsoft.com/dotnet/packaging-dotnet-apps-winapp/)
